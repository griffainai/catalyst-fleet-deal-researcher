# working-theory.md — Live Scaffold

The researcher maintains this file across a session (and across sessions, if logged). It is the *current state* of the thesis for the fleet(s) under investigation. It is meant to be edited, not preserved — a thesis that never changes was never tested.

Empty on first use. The researcher populates it once an angle is set.

---

## Template (one block per fleet)

```
### [Fleet Name] — last updated [date]
ANGLE: [open / qualify / call-prep / walk]
CURRENT CLAIM: [one sentence — may change]

CATALYST: [family + dated event + tier + window status]
SIGNER: [role + name-or-Unknown + confidence]

LEDGER
  Known:    ...
  Inferred: ...
  Unknown:  ... (with how to close each)

KILL-CONDITIONS (still open):
  1. ...
  2. ...
  3. ...

WHAT CHANGED THIS SESSION:
  - [operator confirmed / killed / added ...]

NEXT QUESTION: ...
```

---

## Discipline

- When the operator confirms a fact, move it from Inferred/Unknown to Known and **date it**.
- When the operator kills a kill-condition (proves it false), the thesis *strengthens* — note it. When a kill-condition turns out true, the thesis dies — say so plainly and recommend the new action.
- Never silently drop a prior claim. If the thesis changes, record *what changed and why* under "WHAT CHANGED THIS SESSION." The trail is part of the research.
