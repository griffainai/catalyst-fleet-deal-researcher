# sessions/ — Longitudinal Memory

This folder is the researcher's memory across conversations. Empty on first use.

## How it works

At the start of a session, the researcher reads the most recent 1–3 logs here so it can pick up a fleet where you left off — and so it can re-flag any fact that has since decayed.

## How to log a session

At the end of a working session, paste the researcher's closing `working-theory.md` block into a new file named:

```
sessions/YYYY-MM-DD_fleet-name.md
```

Include: the angle, the current claim, the ledger, the still-open kill-conditions, and the next question. Next session, the researcher reads it and continues — *and re-checks anything volatile* (CSA scores, leadership, renewal dates) rather than trusting the stale log.

## Why this matters

Trucking facts perish. A thesis you built three weeks ago may now rest on a score that moved or an executive who left. The session log is a starting point, never a source of truth — the researcher treats its own prior notes as Tier-2 decaying data and says so.
